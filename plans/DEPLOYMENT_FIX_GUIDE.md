# Deployment Fix Guide for Hawladar Agro Django Project

## Overview

This guide addresses the issues encountered during deployment to the cPanel server and provides solutions to successfully deploy the Django portfolio project.

## Issues Encountered

### 1. Python Version Incompatibility ❌

**Problem:**
- Server has Python 3.9.25 installed
- Django 5.0.1 requires Python 3.10 or higher
- Dependency installation fails due to version mismatch

**Solution Options:**

#### Option A: Request Python 3.10+ Installation (Recommended)
Contact your hosting provider to install Python 3.10 or 3.11 on the server.

**Steps:**
1. Log in to cPanel
2. Go to "Support" or "Contact Support"
3. Request Python 3.10 or 3.11 installation
4. Specify: "Need Python 3.10+ for Django 5.0.1 deployment"

#### Option B: Downgrade Django to Compatible Version
Use Django 4.2.x which supports Python 3.9.

**Steps:**
1. Update `requirements.txt`:
   ```txt
   Django==4.2.11
   djangorestframework==3.14.0
   psycopg2-binary==2.9.9
   Pillow==10.2.0
   django-environ==0.11.2
   ```

2. Update `requirements-prod.txt`:
   ```txt
   # Production requirements for Hawladar Agro Django project
   # Includes base requirements plus production-specific packages

   -r requirements.txt

   # Production WSGI server
   gunicorn==21.2.0

   # Static file serving
   whitenoise==6.6.0

   # Security
   django-cors-headers==4.3.1

   # Email
   django-anymail==10.2
   ```

3. Update `hawladar_agro/settings.py` line 4:
   ```python
   Generated by 'django-admin startproject' using Django 4.2.11.
   ```

#### Option C: Use cPanel Python App Interface (Alternative)
Use cPanel's built-in Python application setup which may have newer Python versions available.

### 2. Corrupted settings_prod.py File ❌

**Problem:**
The `settings_prod.py` file on the server was corrupted with incorrect import statement.

**Solution:**

#### Fix 1: Restore from Git (If Available)
```bash
cd ~/hawladaragro.farm
git checkout HEAD -- hawladar_agro/settings_prod.py
```

#### Fix 2: Recreate the File Manually
Create `hawladar_agro/settings_prod.py` with the following content:

```python
"""
Production settings for hawladar_agro project.
"""
from .settings import *
import os

DEBUG = False

ALLOWED_HOSTS = env.list('ALLOWED_HOSTS', default=['hawladaragro.farm', 'www.hawladaragro.farm'])

# PostgreSQL database for production
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': env('DB_NAME', default='hawladaragro_db'),
        'USER': env('DB_USER', default='hawladaragro_user'),
        'PASSWORD': env('DB_PASSWORD'),
        'HOST': env('DB_HOST', default='localhost'),
        'PORT': env('DB_PORT', default='5432'),
    }
}

# Static files configuration for production
STATIC_ROOT = BASE_DIR / 'public_html' / 'static'

# Media files configuration for production
MEDIA_ROOT = BASE_DIR / 'public_html' / 'media'

# Security settings
SECURE_BROWSER_XSS_FILTER = True
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_HSTS_SECONDS = 31536000
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True

# SSL/HTTPS
SECURE_SSL_REDIRECT = True

# Email configuration
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = env('EMAIL_HOST', default='smtp.gmail.com')
EMAIL_PORT = env('EMAIL_PORT', default=587)
EMAIL_USE_TLS = True
EMAIL_HOST_USER = env('EMAIL_HOST_USER')
EMAIL_HOST_PASSWORD = env('EMAIL_HOST_PASSWORD')
DEFAULT_FROM_EMAIL = env('DEFAULT_FROM_EMAIL', default='noreply@hawladaragro.farm')

# Logging
LOGGING = {
    'version': 1,
    'disable_existing_loggers': False,
    'formatters': {
        'verbose': {
            'format': '{levelname} {asctime} {module} {message}',
            'style': '{',
        },
    },
    'handlers': {
        'file': {
            'level': 'WARNING',
            'class': 'logging.FileHandler',
            'filename': BASE_DIR / 'logs' / 'django.log',
            'formatter': 'verbose',
        },
        'console': {
            'level': 'INFO',
            'class': 'logging.StreamHandler',
            'formatter': 'verbose',
        },
    },
    'loggers': {
        'django': {
            'handlers': ['file', 'console'],
            'level': 'WARNING',
            'propagate': True,
        },
        'portfolio': {
            'handlers': ['file', 'console'],
            'level': 'INFO',
            'propagate': True,
        },
    },
}

# Create logs directory if it doesn't exist
os.makedirs(BASE_DIR / 'logs', exist_ok=True)
```

## Complete Deployment Procedure (Fixed)

### Prerequisites

1. **Database Created:** PostgreSQL database `kalobira_hawladaragro_db` with user `kalobira_hawladaragro_user`
2. **Backup Completed:** WordPress site backed up to `~/backups/wordpress-20260308-001425/`
3. **Project Cloned:** Django project cloned to `~/hawladaragro.farm/`

### Step-by-Step Deployment

#### Step 1: Fix Python Version Issue

**Choose one option:**

**Option A: Request Python 3.10+ from hosting provider** (Recommended)

**Option B: Downgrade Django to 4.2.11** (If Python 3.10+ unavailable)

Update requirements files locally, then commit and push:
```bash
# On your local machine
git add requirements.txt requirements-prod.txt hawladar_agro/settings.py
git commit -m "Downgrade Django to 4.2.11 for Python 3.9 compatibility"
git push origin main
```

Then on server:
```bash
cd ~/hawladaragro.farm
git pull origin main
```

#### Step 2: Verify/Restore settings_prod.py

```bash
cd ~/hawladaragro.farm

# Check if file is corrupted
cat hawladar_agro/settings_prod.py

# If corrupted, restore from git
git checkout HEAD -- hawladar_agro/settings_prod.py

# Or recreate manually (see Fix 2 above)
```

#### Step 3: Create .env File

```bash
cd ~/hawladaragro.farm

# Copy example file
cp .env.example .env

# Edit with production values
nano .env
```

**Update .env with these values:**
```env
# SECURITY SETTINGS
SECRET_KEY=generate-new-secret-key-here
DEBUG=False
ALLOWED_HOSTS=hawladaragro.farm,www.hawladaragro.farm

# DATABASE CONFIGURATION
DB_NAME=kalobira_hawladaragro_db
DB_USER=kalobira_hawladaragro_user
DB_PASSWORD=Zerin4321*
DB_HOST=localhost
DB_PORT=5432

# EMAIL CONFIGURATION (Optional - leave as is for now)
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_HOST_USER=your-email@example.com
EMAIL_HOST_PASSWORD=your-email-password-here
DEFAULT_FROM_EMAIL=noreply@hawladaragro.farm
```

**Generate a new SECRET_KEY:**
```bash
python3 -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

#### Step 4: Setup Python Virtual Environment

```bash
cd ~/hawladaragro.farm

# Remove old venv if it exists
rm -rf venv

# Create new virtual environment
/usr/bin/python3.9 -m venv venv

# Activate virtual environment
source venv/bin/activate

# Upgrade pip
pip install --upgrade pip

# Install dependencies
pip install -r requirements-prod.txt
```

#### Step 5: Run Migrations

```bash
cd ~/hawladaragro.farm
source venv/bin/activate

# Run migrations
python manage.py migrate --settings=hawladar_agro.settings_prod

# Create superuser (optional)
python manage.py createsuperuser --settings=hawladar_agro.settings_prod
```

#### Step 6: Collect Static Files

```bash
cd ~/hawladaragro.farm
source venv/bin/activate

# Create static directory if it doesn't exist
mkdir -p public_html/static

# Collect static files
python manage.py collectstatic --settings=hawladar_agro.settings_prod --noinput
```

#### Step 7: Configure cPanel Python Application

1. Log in to cPanel
2. Go to "Setup Python App"
3. Click "Create Application"
4. Configure:
   - **Python Version:** 3.9 (or 3.10/3.11 if available)
   - **Application Root:** `hawladaragro.farm`
   - **Application URL:** `hawladaragro.farm`
   - **Application Entry Point:** `passenger_wsgi.py`
   - **Application Startup File:** `passenger_wsgi.py`
   - **Passenger Log File:** `logs/passenger.log`
   - **Passenger Error Log File:** `logs/passenger_error.log`

5. Click "Create"

6. After creation, click "Restart" to restart the application

#### Step 8: Configure WSGI File

Ensure `passenger_wsgi.py` exists with correct content:

```python
import os
import sys

# Add project directory to Python path
project_directory = '/home/yourusername/hawladaragro.farm'
if project_directory not in sys.path:
    sys.path.insert(0, project_directory)

# Set Django settings module
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'hawladar_agro.settings_prod')

# Import Django and get WSGI application
from django.core.wsgi import get_wsgi_application
application = get_wsgi_application()
```

**Update `yourusername` with your actual cPanel username.**

#### Step 9: Configure Domain Document Root

1. In cPanel, go to "Domains"
2. Find `hawladaragro.farm`
3. Click "Manage"
4. Set document root to: `hawladaragro.farm/public_html`
5. Click "Save"

#### Step 10: Configure SSL Certificate

1. In cPanel, go to "SSL/TLS Status"
2. Find `hawladaragro.farm`
3. Click "Run AutoSSL"
4. Wait for certificate to be issued

#### Step 11: Test Deployment

1. Visit `https://hawladaragro.farm`
2. Check if site loads correctly
3. Test all pages:
   - Home page
   - About page
   - Projects page
   - Contact page
   - Gallery page
   - Investment page
   - Blog page

4. Check logs if issues occur:
   ```bash
   tail -f ~/hawladaragro.farm/logs/django.log
   tail -f ~/hawladaragro.farm/logs/passenger_error.log
   ```

## Troubleshooting

### Issue: "ModuleNotFoundError: No module named 'django'"

**Solution:**
```bash
cd ~/hawladaragro.farm
source venv/bin/activate
pip install -r requirements-prod.txt
```

### Issue: "django.db.utils.OperationalError: FATAL: password authentication failed"

**Solution:**
1. Check `.env` file has correct database password
2. Verify database credentials in cPanel > PostgreSQL Databases
3. Update `.env` if needed

### Issue: Static files not loading

**Solution:**
```bash
cd ~/hawladaragro.farm
source venv/bin/activate
python manage.py collectstatic --settings=hawladar_agro.settings_prod --noinput
```

### Issue: "DisallowedHost at / Invalid HTTP_HOST header"

**Solution:**
1. Check `.env` file has correct `ALLOWED_HOSTS`
2. Ensure domain is included: `hawladaragro.farm,www.hawladaragro.farm`
3. Restart Python application in cPanel

### Issue: "Permission denied" when creating logs directory

**Solution:**
```bash
cd ~/hawladaragro.farm
mkdir -p logs
chmod 755 logs
```

## Rollback Procedure

If deployment fails and you need to restore WordPress:

### Quick Rollback (Files Only)

```bash
# Backup current Django files
cd ~/hawladaragro.farm
tar -czf ~/backups/django-failed-$(date +%Y%m%d-%H%M%S).tar.gz .

# Restore WordPress files
cp -r ~/backups/wordpress-20260308-001425/public_html/* ~/public_html/

# Restore WordPress database
mysql -u kalobira_hawladaragro_user -p kalobira_hawladaragro_db < ~/backups/wordpress-20260308-001425/database.sql
```

### Full Rollback via cPanel

1. Log in to cPanel
2. Go to "Backup Wizard"
3. Select "Restore"
4. Restore WordPress database from backup
5. Restore WordPress files from backup
